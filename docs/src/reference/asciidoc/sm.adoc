[[statemachine]]
= Using Spring Statemachine

참고 문서의 이 파트는 Spring Statemachine이 다른 어떤 Spring 기반 application에 제공하는 핵심 기능에 대해 설명한다.

그것은 다음 주제를 포함한다.:

* <<sm-config>> : 일반적인 설정 지원을 설명한다.

* <<sm-machineid>> : machine id 사용에 대해 설명한다.

* <<sm-factories>> : 일반적인 state machine factory 지원에 대해 설명한다.

* <<sm-deferevents>> : deferred event 지원에 대해 설명한다.

* <<sm-scopes>> : scope 지원에 대해 설명한다.

* <<sm-actions>> : actions 지원에 대해 설명한다.

* <<sm-guards>> : guard 지원에 대해 설명한다.

* <<sm-extendedstate>> : 확장된 state 지원에 대해 설명한다.

* <<sm-statecontext>> : state context 지원에 대해 설명한다.

* <<sm-triggers>> : triggers 사용에 대해 설명한다.

* <<sm-listeners>> : state machine listeners 사용에 대해 설명한다.

* <<sm-context>> : 일반적인 Spring application context 지원에 대해 설명한다.

* <<sm-accessor>> : state machine 내부 accessor 지원에 대해 설명한다.

* <<sm-interceptor>> : state machine 에러 핸들링 지원에 대해 설명한다.

* <<sm-security>> : state machine security 지원에 대해 설명한다.

* <<sm-error-handling>> : state machine 인터셉터 지원에 대해 설명한다.

* <<sm-service>> : state machine service 지원에 대해 설명한다.

* <<sm-persist>> : state machine persisting 지원에 대해 설명한다.

* <<sm-boot>> : Spring Boot 지원에 대해 설명한다.

* <<sm-monitoring>> : monitoring 과 trancing 지원에 대해 설명한다.

* <<sm-distributed>> : 분산 state machine 지원에 대해 설명한다.

* <<sm-test>> : state machine 테스트 지원에 대해 설명한다.

* <<sm-papyrus>> : state machine UML 모델링 지원에 대해 설명한다.

* <<sm-repository>> : state machine repository 설정 지원에 대해 설명한다.
